Client Request → 2. Filter Chain Invocation → 3. Authentication Filter → 4. Authentication Manager → 5. DaoAuthenticationProvider → 6. Authentication Result → 7. Security Context → 8. Access Control → 9. Request to Controller → 10. Response


1) Ang una ay mag rerequest si client 
2) Papasok ngayon si FilterChain na kung saan nandun si UsernamePasswordAuthenticationFilter
3) Si UsernamePasswordAuthenticationFilter ngayon ang kukuha ng credentials na binigay ni client. (Username and Password), at gagawa sya ng Initial Authentication object para maidala kay Authentication Manager.
4) Ang gagawin naman ni Authentication Manager ay ibibigay nya kay DaoAuthenticationProvider or any AuthenticationProvider si UsernamePasswordAuthenticationToken(Authentication Object) para iprocess ngayon yung authentication logic.
5) Pagkakuha ni Authentication Provider ng object, kukunin ngayon ni Provider yung username from that object and ichecheck nya kay UserDetailsService kung meron ba sa database nung username na nilagay ni client. Pag meron, ichecheck ngayon yung password na ininput ni User at yung nasa database gamit ang BCryptPasswordEncoder. If same ng result, maiseset ngayon yung details ni Username sa UserPrincipal and yun ang kukunin ni Provider at magseset ng bagong Authentication Object or UsernamePasswordAuthenticationToken para ibalik kay Manager.
6) Ang gagawin naman uli ni Manager ay papalitan yung naunang Authentication Object(not authenticated) ng bagong Authentication Object(authenticated) na galing kay Provider at ilalagay ni Manager ito sa SecurityContextHolder.
7) Ang ginagawa ni SecurityContextHolder ay sya ang hahawak ngayon nung authenticated na Authentication Object. Para ngayon dito ppunta yung ibang objects kung authorized ba si User makapag perform ng iba't ibang actions.
8) Sa Access Control naman ang nangyayari ay dito magkakaroon ng decision making, bago sila pumunta sa controllers na nirerequest ni user, chinecheck muna nila yung Authentication Object na na kay SecurityContextHolder kung authorized ba si user gumawa ng actions base sa nirerequest ni user.
9) If nakapasa si User, tsaka ngayon makakapunta si User sa controllers na kung saan nandito lahat ng endpoints and actions na kinakailangan ni client or user.
10) Ang response ngayon ni Controller ay dipende kung anong Controller ang ginamit at kung anong action ang nirequest ni Client. Kung naka RestController, naka JSON format ang response. If naka traditional Controller, view ang response.
